Project Overview

Dr. J. Peter Moore needed a comprehensive digital portfolio to showcase his work as an educator, letterpress artist, and small-press publisher. The challenge was to translate complex print-based content—seminar courses, letterpress projects, and limited-edition publications—into an engaging, accessible online experience that honored his unique artistic sensibility.

What made this project distinctive was the decision to skip traditional mockup tools entirely and design directly in code. Dr. Moore's aesthetic vision—rooted in fine typography, careful spacing, and the tactile qualities of letterpress work—required a level of refinement that Figma couldn't adequately represent. By coding the design from the start, I could iterate with real fonts, authentic browser rendering, and immediate responsive feedback.

I handled everything from initial concept to deployment, creating a fully custom website that reflects the craftsmanship and literary sophistication of Dr. Moore's work.

The Challenge

This project presented a unique creative challenge: designing for someone whose entire career centers on visual refinement, typography, and the materiality of print. Dr. Moore isn't just a client—he's an artist with decades of experience in letterpress printing and book design, with an exceptionally discerning eye for typographic detail.

Core Design Challenges

  • Unique Artistic Vision: Dr. Moore's aesthetic is deeply personal, rooted in traditional printmaking and fine typography
  • Typography-First Approach: Every typeface, size, weight, and spacing decision needed to honor his expertise
  • Beyond Standard Tools: Figma prototypes couldn't capture the nuanced typographic refinement required—real fonts and browser rendering were essential
  • Personal Portfolio Context: This wasn't a commercial site; it needed to reflect his personality and artistic sensibility
  • Tactile to Digital: Translate the physical qualities of letterpress work into digital experience
  • Content Complexity: Balance academic content, artistic projects, and literary publications

Thinking Outside the Box

I made an unconventional decision: skip Figma entirely and design directly in code. Here's why:

  • Figma's font rendering doesn't match real web typography
  • Browser-specific rendering and anti-aliasing matter for letterpress aesthetics
  • Real CSS control over letter-spacing, line-height, and optical sizing was crucial
  • Immediate responsive testing with actual fonts prevented compromises
  • Faster iteration with live code than mockup-to-code translation

Design Process: Coding as Design

The Unconventional Approach

With the vision board established, I made a critical methodological decision: instead of creating detailed Figma mockups, I would design directly in code. This wasn't about taking shortcuts—it was about achieving the level of typographic refinement the project demanded.

Why Skip Figma?

  • Font Rendering Accuracy: Web fonts render differently in browsers than in Figma; real browser testing was essential
  • Typographic Control: CSS gave me precise control over letter-spacing, line-height, optical sizing, and font features
  • Responsive Typography: Testing actual responsive behavior with real fonts prevented design compromises
  • Iteration Speed: Changing a font stack or spacing value in CSS was faster than updating across multiple Figma frames
  • Client Review: Dr. Moore could see the actual site, not an approximation
  • Anti-Aliasing: Browser rendering and subpixel positioning matter for letterpress-quality aesthetics
/* Example: Fine-tuning typography in CSS */ h1 { font-family: 'Garamond Premier Pro', 'Cormorant Garamond', serif; font-size: clamp(2.5rem, 5vw, 4rem); font-weight: 400; letter-spacing: -0.02em; /* Optical adjustment */ line-height: 1.2; font-feature-settings: 'liga' 1, 'kern' 1, 'onum' 1; /* OpenType features */ }

The Workflow

  • Step 1: Rough HTML structure and content hierarchy
  • Step 2: Typography system with multiple typeface tests
  • Step 3: Layout grid with CSS Grid/Flexbox
  • Step 4: Spacing and whitespace refinement
  • Step 5: Responsive breakpoints with live device testing
  • Step 6: Review sessions with Dr. Moore on actual browsers
  • Step 7: Iteration based on typographic nuance feedback

Design Process

Skipping Figma: Coding as Design

Given Dr. Moore's unique aesthetic vision—one that emphasized subtle typographic details, precise spacing, and the feeling of a well-crafted book—I realized that Figma mockups would be an inadequate intermediary. The nuances of web fonts, actual browser rendering, and responsive behavior needed to be felt and refined in their native environment.

I made the strategic decision to design directly in code, creating a live prototype that we could iterate on together. This approach allowed for:

  • Real-time typography adjustments with actual web fonts
  • Authentic spacing and rhythm impossible to replicate in static mockups
  • Immediate testing of responsive behavior across devices
  • Faster iteration cycles with instant client feedback
  • More accurate representation of the final experience
/* Example: Custom Typography System */ :root { --font-serif: 'Cormorant Garamond', Georgia, serif; --font-sans: 'Inter', -apple-system, sans-serif; /* Modular scale for typography */ --scale-ratio: 1.333; --text-base: 1.125rem; --text-sm: calc(var(--text-base) / var(--scale-ratio)); --text-lg: calc(var(--text-base) * var(--scale-ratio)); --text-xl: calc(var(--text-lg) * var(--scale-ratio)); }

Key Learnings

  • Design Tools are Means, Not Ends: Recognizing when to bypass traditional tools like Figma in favor of direct coding enabled more authentic design expression
  • Code as Design Medium: Designing in the browser allowed for nuanced control over typography, spacing, and responsive behavior that mockups couldn't capture
  • Client Collaboration: Working with live prototypes enabled more productive feedback sessions focused on actual experience rather than static representations
  • Content First: Deep understanding of Dr. Moore's work and aesthetic informed every design decision
  • Performance Matters: Image optimization crucial for image-heavy portfolio sites
  • Responsive Reality: Testing on real devices revealed subtleties impossible to anticipate in design tools
  • Artistic Vision Requires Flexibility: Being willing to adapt process to serve unique creative visions leads to better outcomes
  • Typography is Everything: For literary portfolios, typographic refinement makes or breaks the experience
